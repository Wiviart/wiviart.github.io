      Properties and Encapsulation

      Before we start to explain properties, you should have a basic understanding of "Encapsulation".

      The meaning ofÂ Encapsulation, is to make sure that "sensitive" data is hidden from users. To achieve this, you must:

      • declare fields/variables asÂ private
      • provideÂ publicÂ getÂ andÂ setÂ methods, throughÂ properties, to access and update the value of aÂ privateÂ field

      Properties

      You learned from the previous chapter thatÂ privateÂ variables can only be accessed within the same class (an outside class has no access to it). However, sometimes we need to access them - and it can be done with properties.

      A property is like a combination of a variable and a method, and it has two methods: aÂ getÂ and aÂ setÂ method:

      Example explained

      TheÂ NameÂ property is associated with theÂ nameÂ field. It is a good practice to use the same name for both the property and the private field, but with an uppercase first letter.

      TheÂ getÂ method returns the value of the variableÂ name.

      TheÂ setÂ method assigns aÂ valueÂ to theÂ nameÂ variable. TheÂ valueÂ keyword represents the value we assign to the property.

      If you don't fully understand it, take a look at the example below.

{

  {

      Abstract Classes and Methods

      DataÂ abstractionÂ is the process of hiding certain details and showing only essential information to the user.Abstraction can be achieved with eitherÂ abstract classes or interfaces (which you will learn more about in the next chapter).

      TheÂ abstractÂ keyword is used for classes and methods:

      Another way to achieveÂ abstractionÂ in C#, is with interfaces.

      AnÂ interfaceÂ is a completely "abstract class", which can only contain abstract methods and properties (with empty bodies):

      By default, members of an interface areÂ abstractÂ andÂ public.

      Note:Â Interfaces can contain properties and methods, but not fields.

      To access the interface methods, the interface must be "implemented" (kinda like inherited) by another class. To implement an interface, use theÂ :Â symbol (just like with inheritance). {

}

{

{
  static void Main(string[] args)
  {
    Pig myPig = new Pig();  // Create a Pig object

  }
}

      Notes on Interfaces:

      • LikeÂ abstract classes, interfacesÂ cannotÂ be used to create objects (in the example above, it is not possible to create an "IAnimal" object in the Program class)
      • Interface methods do not have a body - the body is provided by the "implement" class
      • On implementation of an interface, you must override all of its methods
      • Interfaces can contain properties and methods, but not fields/variables
      • Interface members are by defaultÂ abstractÂ andÂ public
      • An interface cannot contain a constructor (as it cannot be used to create objects)
